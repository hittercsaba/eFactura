API Documentation
Base URL
{{ request.url_root.rstrip('/') }}/api/v1
Authentication

Include your API key in the X-API-KEY header for all requests:

X-API-KEY: your-api-key-here
List Invoices
Endpoint
GET /api/v1/invoices
Query Parameters
  • page - Page number (default: 1)
  • per_page - Items per page (default: 50, max: 100)
  • supplier_cif - Filter by supplier CIF (optional)
  • date_from - Filter from date, format: YYYY-MM-DD (optional)
  • date_to - Filter to date, format: YYYY-MM-DD (optional)
Example Request
curl -H "X-API-KEY: your-api-key-here" \
  "{{ request.url_root.rstrip('/') }}/api/v1/invoices?page=1&per_page=50"
Response Fields

Each invoice in the response includes:

  • id - Internal invoice ID
  • anaf_id - ANAF message identifier
  • invoice_type - "FACTURA PRIMITA" (received) or "FACTURA TRIMISA" (sent)
  • cif_emitent - Issuer CIF
  • cif_beneficiar - Receiver CIF
  • issuer_name - Issuer company name
  • receiver_name - Receiver company name
  • invoice_date - Invoice date (YYYY-MM-DD)
  • total_amount - Total invoice amount
  • synced_at - Sync timestamp
  • download_url - URL to download invoice ZIP file
Get Invoice by ID
Endpoint
GET /api/v1/invoices/{invoice_id}
Example Request
curl -H "X-API-KEY: your-api-key-here" \
  "{{ request.url_root.rstrip('/') }}/api/v1/invoices/1"

Returns full invoice details including parsed JSON content.

Download Invoice ZIP
Endpoint
GET /api/v1/invoices/{invoice_id}/download
Description

Downloads the invoice as a ZIP file containing the XML invoice file(s). The ZIP typically contains:

  • The invoice XML file (UBL format)
  • A signature XML file (if available)
Example Request
curl -H "X-API-KEY: your-api-key-here" \
  "{{ request.url_root.rstrip('/') }}/api/v1/invoices/1/download" \
  -o invoice.zip
Response
  • Content-Type: application/zip
  • Content-Disposition: attachment; filename=invoice_{anaf_id}.zip
  • Body: Binary ZIP file content
Note: The download URL is automatically included in each invoice object in the API response. Use the same X-API-KEY header when accessing the download URL.
Error Responses

All errors follow a consistent format:

{
  "error": "Error Type",
  "message": "Human-readable error message",
  "code": "ERROR_CODE"
}
HTTP Status Codes
  • 200 - Success
  • 400 - Bad Request (invalid parameters)
  • 401 - Unauthorized (missing or invalid API key)
  • 404 - Not Found (resource doesn't exist or not accessible)
  • 429 - Too Many Requests (rate limit exceeded)
  • 500 - Internal Server Error
